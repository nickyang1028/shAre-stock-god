# Stock Fullstack Builder

## 概览

使用这个 skill 时，按“共享类型 -> 后端能力 -> 前端消费 -> 验证”的顺序推进，避免前后端各自定义一套数据形状。

## 开发流程

1. 先确认 monorepo 结构和包管理器，优先沿用项目已有约定。
2. 把跨端类型放在 `packages/shared` 或项目已有共享模块里。
3. 后端接口返回稳定 JSON，字段使用英文枚举，展示文案交给前端或共享常量。
4. 前端通过 service/client 层请求接口，不在组件里拼散乱 URL。
5. 图表、信号列表、详情面板拆成清晰组件，避免一个页面文件承载全部业务逻辑。
6. 功能完成后至少运行类型检查；涉及规则或接口时补单元测试。

## 前端约定

- 使用 TypeScript 严格建模行情、指标和信号数据。
- K 线图组件只负责绘制，不负责计算指标或识别信号。
- 信号标记需要有稳定 `id`，避免 React 列表和图表 overlay 抖动。
- UI 文案避免承诺收益，使用“偏多”“偏空”“中性”“风险提示”等表达。

## 后端约定

- 行情获取、指标计算、信号识别分模块实现。
- API 参数需要校验：`symbol`、`period`、`limit`、指标名、时间范围。
- 错误响应保持统一结构，方便前端展示。
- 指标与信号计算函数保持纯函数，便于测试和后续回测复用。
