# Kline Signal Engineer

## 概览

使用这个 skill 时，把每一个交易信号都当成“可解释的行情观察”，不要把信号写成投资建议或买卖承诺。

## 工作流程

1. 先查找已有共享类型，优先复用 `KLine`、`Signal`、`SignalType` 和指标类型。
2. 把信号识别逻辑写成纯函数，放在后端或共享领域层，避免散落在 React 组件里。
3. 每个信号规则单独实现，输入有序 K 线或指标序列，输出 `Signal[]`。
4. 信号只能使用当前及历史数据，禁止使用未来数据，避免回测里的未来函数问题。
5. 为每个规则补单元测试，覆盖数据不足、平盘、跳空、零成交量、极端长影线等边界情况。

## 规则约定

- 默认 K 线按时间升序排列。
- 阳包阴、阴包阳优先按实体包覆判断，除非需求明确要求按最高价/最低价全包覆。
- 十字星、锤子线、射击之星等模糊形态必须使用可配置阈值。
- 金叉、死叉必须比较前一根与当前根，不能只看当前大小关系。
- `strength` 保守评分：只有趋势、成交量、指标确认同时出现时才给高分。
- `description` 只描述发生了什么，不写“必涨”“必跌”等结论。
